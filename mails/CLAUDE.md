# mails/ — 베데스다 담당자 메일 아카이브

> 루트 `../CLAUDE.md` 먼저. 이 파일은 mails/ 전용 지침.

## 역할
베데스다 iCReaT DCT 담당자와 메일로 주고받은 **내용·합의·스펙**의 1차 출처 보관소.
업로드 파이프라인 식별자(`studyId`/`subjectId`), 필드명, 엔드포인트, 멀티파트 규격, 기한 등 임상 데이터 무결성이 걸린 결정은 여기서 근거 확인 후 코드 반영. 추측 금지.

## 폴더 구조 (타입별 분류)
```
mails/
├─ CLAUDE.md          이 지침
├─ CHANGELOG.md       아카이브 (메일 추가·갱신 이력)
├─ threads/           메일 본문 문서화 (.md) — 한 주제 = 한 파일, 시간순 누적
└─ attachments/       주고받은 원본 파일 (zip/pdf/xlsx/이미지 등)
```

## 네이밍 규칙
- 스레드: `threads/YYYY-MM-DD_<주제-kebab>.md`
  - 날짜 = **스레드 시작일**(첫 메일 날짜). 같은 주제 후속 메일은 같은 파일에 시간순 append.
  - 주제는 짧은 영문/한글 kebab. 예: `2026-06-30_subject-id-mapping.md`, `2026-06-30_csv-multipart-spec.md`
- 첨부: `attachments/YYYY-MM-DD_<원본파일명>`
  - 날짜 = 그 파일을 주고받은 메일 날짜. 원본 파일명 보존(확장자 유지).
  - 예: `attachments/2026-06-30_iCReaT_DCT_sensor_upload.zip`

## 스레드 .md 문서 양식 (필수)
각 `threads/*.md` 상단에 프론트매터, 본문은 메일 단위 블록으로 시간순.
```markdown
---
주제: <한 줄 요약>
상대: <담당자 이름/역할/메일주소>
상태: 진행중 | 합의됨 | 보류 | 종료
첨부: attachments/2026-06-30_xxx.zip (있으면)
---

## 2026-06-30 ← 받음 (또는 → 보냄)
<메일 본문 핵심. 원문 인용 시 > 블록. 합의/요청사항은 굵게>

**결정/액션:** <코드·보고에 반영할 사항>

---

## 2026-07-02 → 보냄
...
```

## 라우팅 (어디서 무엇을 찾나)
- 코드 작업 중 "베데스다가 뭐라 했더라" → `threads/` 해당 주제 파일에서 **결정/액션** 줄 확인.
- **발표자료(`reports/`)는 메일 원문을 복사하지 않는다.** threads/는 원문 전체 보관소, 슬라이드는 거기서 **사실 1개**만 뽑아 결론·수치로 distill. 메일 인사말·따옴표 인용 슬라이드 박제 금지 (상세: `reports/CLAUDE.md` 슬라이드 내용 규칙).
- 보고서(`reports/notes/bethesda_*.md`)는 분석·요약 산출물, mails/는 **원본 근거**. 둘이 충돌하면 mails/가 1차.
- 첨부 원본은 항상 `attachments/`. threads/ 본문에서 상대경로로 링크.

## 정리 규칙 (필수)
- mails/ 루트에 .md 본문/첨부를 흩뿌리지 마라 — threads/ 또는 attachments/로.
- `CLAUDE.md`, `CHANGELOG.md`만 루트에 둔다.
- 새 스레드/첨부 추가 시 `CHANGELOG.md`에 항목 추가, 위 "폴더 구조"가 바뀌면 표도 갱신.

## 보안 (필수)
- 메일에 담긴 **개인식별정보(연구대상자 실명·연락처·주민번호 등)는 문서화 시 마스킹**. 임상 데이터 무결성·개인정보 보호.
- 인증 토큰·API 키·비밀번호가 메일에 있으면 threads/에 옮기지 말 것 — `pbcr_source/key/`와 동일하게 git 제외 대상. 필요 시 "키는 별도 보관" 메모만.
- 첨부에 시크릿 포함 시 커밋 전 `git status`로 확인.

## 아카이브 규칙 (필수)
이 폴더에 메일/첨부 추가·갱신 시:
- `CHANGELOG.md`에 항목 추가 (형식은 그 파일 헤더 참조).
- 코드 변경을 유발한 메일은, 코드 쪽 폴더 `CHANGELOG.md` 인라인 주석에서 이 스레드 파일을 근거로 링크.
